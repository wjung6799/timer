# Bootstrapped Capacitor iOS with Apple compliance baseline

## What Changed
Commit ff7a2aa. Capacitor 8 added with iOS platform, bundle id com.budget.timebudget. Plugins: local-notifications, app, browser, preferences. Notifications scheduled in App.tsx on Start/Pomo for budget-reached + pomo-end and cancelled on Stop/remove/complete/sign-out/auto-stop (notify.ts wraps and no-ops on web). OAuth deep-link handler at top-level App component parses budgetapp://callback (both implicit and PKCE flows) and closes Browser. Auth.tsx adds "Continue with Apple" required by Apple guidelines because Google is offered, and uses Browser.open with skipBrowserRedirect on native. Info.plist has CFBundleURLTypes for budgetapp, NSUserTrackingUsageDescription (no tracking), ITSAppUsesNonExemptEncryption=false, UIBackgroundModes remote-notification. PrivacyInfo.xcprivacy declares no tracking, app-functionality only, CA92.1 reason for @capacitor/preferences UserDefaults access. Privacy policy at public/privacy.html. npm scripts ios:sync and ios:open added. CLAUDE.md documents the build flow.</changes>
<parameter name="decisions">Used skipBrowserRedirect+Browser.open for OAuth on native rather than the native ASAuthorizationController because it works for both Apple and Google with the same code path and doesn't require a separate Apple Sign In plugin. Sign in with Apple is required for App Store approval since Google is offered. Privacy policy hosted as a static HTML file in public/ rather than a React route to avoid coupling and to keep it accessible without bundling.</decisions>

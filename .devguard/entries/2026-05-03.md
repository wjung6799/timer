# Pre-deploy commit: auth-optional + Settings page + weekday defaults + cross-midnight timer

## What Changed
Squash commit 81c03a1 on master. Sign-in now optional (localStorage fallback, migrates to Supabase on first sign-in). Settings is a routed page (sibling of History) with default categories editor, per-weekday overrides via tabs, Reset Today, Clear All Data, and Delete Account (via security-definer delete_user() RPC). Per-category pomodoro length editable inline. Day rollover (live-tick + load-time) caps yesterday at midnight and carries an active timer onto today's name-matched category — so a Sleep timer keeps tracking across midnight. Tab-close saves state without committing the active timer (off-tab elapsed counts on reopen). Added .env.example for deploys.</changes>
<parameter name="decisions">Use Supabase as the entire backend (no separate API server) — RLS isolates per-user data, publishable key is safe to bundle, account deletion runs via a security-definer RPC. Per-weekday defaults stored on AppState as a Partial&lt;Record&lt;Weekday, DefaultCategory[]&gt;&gt; alongside an "All days" base, with inheritance: missing key falls back to base. Cross-midnight timer carryover only applies when today is exactly the day after the saved date, to avoid auto-restarting a timer after a multi-day gap.</decisions>
<parameter name="next_steps">Deploy via Vercel: user is running `npx vercel` interactively, then env vars (VITE_SUPABASE_URL, VITE_SUPABASE_PUBLISHABLE_KEY), then `vercel --prod`. After prod URL is known, add it to Google Cloud OAuth Authorized JavaScript origins and to Supabase Auth → URL Configuration (Site URL + redirect allow list). Supabase callback URL stays the same.

# Pivoted the app into a "productivity hub": added react-router, a Hub landing, a new Timers module (visual countdown + task checklist), and moved Time Budget to /budget.

## What Changed
- Added react-router-dom. New entry chain: main.tsx → BrowserRouter → src/Root.tsx (owns auth bootstrap + OAuth deep-link handler) → Routes: / = Hub, /timers = TimersApp, /budget = BudgetApp, * → /.
- src/App.tsx: removed the old default App wrapper; BudgetApp is now a named export; deep-link effect moved to Root; added a "← Hub" link in its header. Added .header-left to App.css.
- New src/hub/Hub.tsx + Hub.css: card grid (Timers, Time Budget, plus "soon" placeholders).
- New src/timers/: types.ts (TimerPreset/TaskItem/ActiveTimer/TimersState + helpers), db.ts (Supabase timers_state table + localStorage fallback, local→remote migration on first sign-in), TimersApp.tsx (presets list, start/pause/resume/stop/+1min, task checklist, time's-up banner, mute), VisualTimer.tsx (SVG Time-Timer-style shrinking pie wedge + digital readout + rim ticks), Timers.css.
- src/notify.ts: added scheduleTimerEnd / cancelTimerNotification (single fixed notification id).
- src/db.ts wipeAllData now also clears timers_state / timers.state.
- docs/supabase-schema.sql: added timers_state table + RLS owner policy. Applied the migration to the live Supabase project (add_timers_state) — it's additive (create table if not exists + RLS).
- vercel.json: SPA rewrite to /index.html. index.html title → "Hub". CLAUDE.md: documented routes/structure; moved the deep-link note to Root.tsx.
- Verified: npm run build passes; smoke-tested /, /timers (started a timer, disk renders), /budget in a browser — no console errors.

## Decisions
- Single Vite app + react-router (path routes), not a true DNS subdomain or separate build — user wants a hub; /budget is the "subdomain" equivalent. Can add a Vercel rewrite from budget.<domain> → /budget later if DNS is set up.
- Timers module reuses the existing auth + Supabase sync pattern (new timers_state table mirroring app_state), not local-only.
- Visual timer is an SVG pie wedge that shrinks clockwise (fraction = remaining/duration) with a digital readout — simplest faithful "Time Timer" look.
- One running timer at a time; active timer state is persisted in TimersState (endsAt, or pausedRemainingSec when paused) so it survives reload.

## Issues
- npm run lint still reports the same ~7 react-hooks v7 strict-rule errors that already exist on main (refs-during-render, setState-in-effect, Date.now purity) plus the pre-existing types.ts:281 'consumed' unused; build is the deploy gate and passes. Left as-is to match existing code patterns.
- Capacitor: iOS uses BrowserRouter served from capacitor://localhost — fine for client nav and the OAuth deep-link, but dist needs `npm run ios:sync` again before the next iOS build.
